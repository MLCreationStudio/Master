## Story
**As a** maintainer do foundo-app,
**I want** que o build da aplicação não quebre quando `OPENAI_API_KEY` estiver ausente,
**so that** eu possa gerar build e deploy com falha controlada apenas nas rotas de IA em runtime.

## Acceptance Criteria
1. `npm run build` conclui com sucesso mesmo sem `OPENAI_API_KEY` definida.
2. A rota `foundo-app/src/app/api/ai/icebreaker/route.ts` não instancia nem avalia o client OpenAI durante o build.
3. A rota `foundo-app/src/app/api/ai/pitch-refiner/route.ts` segue o mesmo padrão técnico da rota `icebreaker`.
4. Quando `OPENAI_API_KEY` estiver ausente em runtime, ambas as rotas de IA retornam erro HTTP 500 com mensagem controlada.
5. A UI que consome `pitch-refiner` trata resposta HTTP de erro sem quebrar a renderização.
